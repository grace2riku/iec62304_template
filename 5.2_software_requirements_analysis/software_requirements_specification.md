## 4. ソフトウェア要求事項(箇条 5.2.2)

各要求事項は以下の項目を含む:
- 一意の ID(`SRS-xxx`)
- 要求の記述
- 優先度(必須 / 推奨)
- 検証方法(試験 / レビュー / 解析 / デモ)
- 関連システム要求・リスクコントロール手段への紐付け

## 7. ソフトウェア要求事項の更新(箇条 5.2.5)

システム要求事項が更新された場合、本 SRS を更新し、変更の影響を関連ドキュメントに波及させる。変更手順は構成管理計画に従う。

## 8. 要求事項の検証(箇条 5.2.6)

各要求事項は以下の基準を満たすことをレビューで確認する:
- システム要求事項への実装可能性
- 相互に矛盾しないこと
- 明確で、試験可能であること(曖昧な表現・主観的表現を含まないこと)
- 一意に識別可能であること
- 医療機器のリスクコントロール手段の実装に使用できること

レビュー記録は構成管理下に保管する。
